# Swag Pet V1 需求文档

## Summary

Swag Pet V1 是一个 **macOS 优先的本地桌面电子宠物应用**。第一版目标是验证“宠物在桌面陪伴用户”的核心体验，不接入后端，不直接接入 DeepSeek；先完成桌面显示、动画状态、基础互动、本地存储和宠物资源规范，为后续 Java Spring AI + DeepSeek 接入预留接口。

V1 技术默认选择：**Electron + React + TypeScript**。宠物形态默认选择：**像素风 / 帧动画**。数据默认保存在本地。

## Key Requirements

- 桌面宠物窗口
  - 显示一个透明背景、无边框、置顶的宠物窗口。
  - 支持鼠标拖拽移动宠物。
  - 支持宠物待机、走动、睡觉、开心、被点击等基础动画状态。
  - 支持托盘菜单：显示/隐藏宠物、重置位置、退出应用。
  - V1 优先适配 macOS，后续再扩展 Windows/Linux。

- 宠物互动
  - 点击宠物后触发反馈动作，如开心、说一句预设话、短暂停留。
  - 支持基础养成属性：心情值、精力值、亲密度、陪伴天数。
  - 支持简单行为循环：长时间无操作后进入待机/睡觉，互动后恢复活跃。
  - 不实现复杂小游戏、商城、账号、好友访问、排行榜。

- 本地存储
  - 保存宠物位置、当前宠物、养成属性、设置项、首次启动时间、最后互动时间。
  - 应用重启后恢复宠物状态和位置。
  - V1 可使用 `electron-store` 或本地 JSON；不引入数据库。

- 宠物资源规范
  - V1 内置 1 只默认像素宠物。
  - 定义资源包结构，为后续用户自定义宠物做准备。
  - 每只宠物包含 `pet.json` 和动画帧资源目录。
  - V1 不做完整资源包导入 UI，但代码结构应允许后续添加。

- AI 预留
  - V1 不直接接入 DeepSeek，不实现 Java 后端。
  - 预留 `PetBrain` 抽象接口，用于后续接入规则引擎、Spring AI 后端或本地模型。
  - V1 的宠物回复使用本地预设语句和规则状态机生成。

## Interfaces And Data

- 宠物资源包最小结构：

```text
pets/default/
  pet.json
  animations/
    idle/
    walk/
    sleep/
    happy/
    tap/
```

- `pet.json` 最小字段：

```json
{
  "id": "default-pet",
  "name": "Swag Pet",
  "version": "1.0.0",
  "style": "pixel",
  "animations": {
    "idle": { "fps": 6, "loop": true },
    "walk": { "fps": 8, "loop": true },
    "sleep": { "fps": 4, "loop": true },
    "happy": { "fps": 8, "loop": false },
    "tap": { "fps": 10, "loop": false }
  }
}
```

- 本地状态最小字段：

```json
{
  "petId": "default-pet",
  "position": { "x": 1200, "y": 720 },
  "mood": 80,
  "energy": 70,
  "affinity": 1,
  "createdAt": "ISO_TIME",
  "lastInteractionAt": "ISO_TIME",
  "settings": {
    "alwaysOnTop": true,
    "launchAtLogin": false,
    "soundEnabled": false
  }
}
```

- 后续 Java 后端预留接口，不在 V1 实现：
  - `POST /api/pet/chat`
  - `GET /api/pet/state`
  - `POST /api/pet/action`
  - `WS /ws/pet-events`

## Acceptance Criteria And Tests

- 桌面体验
  - 启动应用后，宠物出现在桌面上，背景透明，无普通窗口边框。
  - 宠物窗口默认置顶，不遮挡为大块白底窗口。
  - 用户可以拖动宠物，重启后位置保持。
  - 托盘菜单可以隐藏、显示、重置位置、退出。

- 动画与互动
  - 默认宠物能播放待机动画。
  - 点击宠物后，能切换到反馈动画并显示一句本地预设文本。
  - 一段时间无互动后，宠物能自动进入待机或睡觉状态。
  - 状态切换不会导致窗口尺寸跳动或动画闪烁。

- 存储
  - 首次启动时生成默认状态。
  - 重启后恢复宠物位置、亲密度、心情值等本地状态。
  - 本地状态文件损坏时，应用能回退到默认状态，不崩溃。

- 资源规范
  - 默认宠物资源通过 `pet.json` 加载。
  - 缺失某个动画时，应用回退到 `idle`，并记录错误日志。
  - 替换默认资源目录后，开发者可以按相同规范添加新宠物。

## Assumptions

- V1 只做本地单机应用，不做账号、云同步、后端、付费、社区资源市场。
- V1 以 macOS 为主要验收平台，Electron 方案优先于 Tauri，以降低早期开发成本。
- V1 使用像素帧动画，不使用 Live2D、Spine、VRM 或 3D 模型。
- V1 的“智能”来自本地规则和预设话术，Spring AI + DeepSeek 放到 V2。
- 项目开源，后续可以参考 GPL/AGPL 项目的设计，但直接复用代码时需保持许可证兼容。
